与猫有关的三则

一

我说#姣姣#有些地方跟猫弟小辉很像，比如最喜欢厨房，最爱找好吃的等等。

最近明俊也说姣姣跟小辉有些像，大概是指都没什么追求这点吧。

二
我说家里四位活物，第一快乐的应该是猫弟小辉，第二快乐的是姣姣。

明俊对姣姣说：你妈妈最不快乐，因为她要求多。

我很诧异的是，明俊认为他比我快乐。我本来想把自己排家里第三快乐的。


三
猫弟小辉最喜欢围着明俊转，我对明俊说："小辉最喜欢你，是因为听不懂你说话。"

姣姣没明白什么意思。

明俊解释说："你和你妈妈能听懂我说话，所以就不喜欢我。(我一说话就会得罪人)"

明俊问我："你和姣姣放在跟猫一样的水平上，不是太那个了么？"

我："猫一直被视作女性的，虽然咱家小辉是公猫"

明俊："我没听说过"

切~，孤陋寡闻。

    "软件定义世界"，几年前@蒋涛CSDN 也说过类似话。
      @硅谷王川:
      世界的运作正在朝软件化的方向演进，这有几个步骤:

      第一，各个节点之间通讯时间延迟压缩到接近于零。

      第二，各个节点之间通讯，交通成本不断压低，无限接近于零。

      第三，某个高效的自发的组织涌现出一个开放的，安全的，有扩展性的应用程序界面 API，可供外界调用。

      第四，使用 API 的客户数越来越多，可以分摊调用程序的费用，使它无限接近于零。

      第五，当所有的延迟和费用都压得很低的时候，无数新程序员可以在新的基础设施上轻松开发各种新的应用程序，迅速淘汰老的做事方法。

      第六，开发应用软件的很多人，其实技术上不一定特别硬核，对一些底层细节可能懂得也并不多，但是最大的利益往往被少数控制应用软件的人获取了。

      第七，软件定义世界没有止境，因为总是还有某些行为需要人工干预，或者通讯运输成本过高，对于这些瓶颈的突破，就意味着新的软件工具的增加，和新的应用软件的涌现。

      亚马逊作为一个公司的运营方式，就是软件定义世界的一个雏形。但这只是刚开始，未来的发展趋势，凡是涉及需要人工干预的行为，都必将一个个被自动化，日常生活中各种服务，交易等等都将成本被不断压缩而且软件化。

      无法彻底软件化的系统，最终必定会因为低效和高成本被迅速淘汰。

      试图解释软件化的概念有时非常困难，就像你无法给一个几十年前习惯珠算的人，去解释 Excel spreadsheet, 去解释各种用于模拟计算的软件工具，等等。

        #创业#

        在尝试做视频前，有一个问题困扰了我很长时间：是做竖版的小视频，还是做横版的短视频？毕竟精力有限，我习惯把一个东西尝试到感觉到玩明白之后，再去尝试另外的形式。

        我还是先选择了竖版的小视频，基于对人性懒惰的认知，刷手机的人习惯是竖着持手机的，让他转动一下手机，以方便看横版的全屏，可能都会影响刷刷刷的愉悦感。我刷那些横板竖版混杂的视频流时，极少会去转动手机。

        但是发现很多流量大的平台还是更支持横版视频，比如微博，只有横版视频才允许推荐给各个频道。一些历史更早的APP，比如美团，也是对横版视频支持得更好。我在琢磨横版和竖版的视频区分在哪里？

        现在我认为区别是在时间长度上，一到三分钟的内容适合做竖版的手机原生视频，因为用手机看视频，三分钟貌似是大部分用户接受的极限。

        但横版视频就可以做到10几分钟甚至半小时以上，因为我们之前习惯的电影和电视都是横版视频。这种多年培养起来的用户使用习惯和场景，也是一时难以改变的。

                        #创业#

                        我开始做电商是2015年，其实那个时候做电商的时机已经过去了，不过因为启动时选在了微信平台，还是拿到了一些流量的。2016年到2018年之间也是微商很猖狂的时候，我研究过他们的玩法，发现自己做不了，因为他们并不是卖产品，而是在卖发财机会。

                        现在在淘宝和京东这样的平台上做电商，只有大资金有品牌的厂商才有机会。小玩家们都不要想太多。而微商现在也有正规军进入，开始真正发挥社交电商的威力。但这个绕不过去社交平台的各种限制和规定，所以社交电商怎么玩儿，还需要拭目以待。玩得好的几位，是接受了大社交平台投资玩家，至于小玩家想去蹭这些机会，草根或许还能挖些墙角，想做大也是属于想多了。

                        所以现在草根想要网络卖货，还只能看短视频平台了，目前头部两家也都在纷纷启动自己的小程序电商，进入要趁早。

                        这个时候愈发要看个体效率。一个人，可以顶10个人，这不仅仅是程序员行业的规律。传统土老板还喜欢招那些只是听话和按指令干活员工的话，玩不转这些。
